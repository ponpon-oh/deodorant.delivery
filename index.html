対応シート形式：横持ち＝1行目が「(タイムスタンプ)｜到着日｜商品A｜商品B｜…」で各セルに個数（＝あなたのシート形式）／ 縦持ち＝「到着日｜商品｜個数」。列見出しの商品名は商品マスタと完全一致させてください。

Google Apps Script中継で同期（CORS/プロキシ不要・最も確実）

Apps Scriptがサーバー側でシートを読み、JSONPで返します。<script>タグ経由で受け取るため、CORSの壁もプロキシも受けず、file://のままでも確実に同期できます。あなたのシート専用コードを用意済みです。

▶ セットアップ手順（コピペ用コード付き）
  1. script.google.com で新規プロジェクトを作成。
  2. 下の「コードをコピー」で全文をコピーし、エディタに貼り付け（シートID・GIDはあなたのシートで設定済み）。
  3. 「デプロイ → 新しいデプロイ → 種類:ウェブアプリ」。実行するユーザー=自分、アクセスできるユーザー=全員 にしてデプロイ。初回は権限承認が出るので許可。
  4. 表示された /exec のURLを上の欄に貼って「保存」→「🔄同期」。到着日・商品・個数が在庫に反映されます。
  5. 「アプリを開いた時に自動で同期する」をONにしておけば、開くたびにこのGAS経由で自動取得します。
function doGet(e){
  var SHEET_ID = '1vqkIYdeaS_zfxq69e0GLdy9QqxeQvLmBhaMB58K-waE';
  var GID = 319471226;
  var ss = SpreadsheetApp.openById(SHEET_ID);
  var sheet = null, all = ss.getSheets();
  for (var i = 0; i < all.length; i++) { if (all[i].getSheetId() === GID) { sheet = all[i]; break; } }
  if (!sheet) sheet = all[0];
  var values = sheet.getDataRange().getValues();
  var tz = ss.getSpreadsheetTimeZone();
  var rows = values.map(function(row){
    return row.map(function(v){
      if (Object.prototype.toString.call(v) === '[object Date]') {
        return Utilities.formatDate(v, tz, 'yyyy-MM-dd HH:mm:ss');
      }
      return v;
    });
  });
  var out = JSON.stringify({ rows: rows });
  var cb = e && e.parameter && e.parameter.callback;
  if (cb) {
    return ContentService.createTextOutput(cb + '(' + out + ')')
      .setMimeType(ContentService.MimeType.JAVASCRIPT);
  }
  return ContentService.createTextOutput(out)
    .setMimeType(ContentService.MimeType.JSON);
}

到着履歴

発注 / 出荷入力（表形式）

縦＝販売単位（あいうえお順、セットも含む）、横＝販路の表です。出荷したいセルに個数だけ入力してください。販売単価と発送方法は販売管理マスタから自動適用されます。「出荷を登録してLINE送信」で在庫差し引きとLINE発注依頼が同時に行われます。

LINEには「発注依頼」＋日付＋「縦=商品／横=発送方法／セル=個数」の表が送られます（販路・価格は含みません）。

出荷履歴

在庫一覧（先入先出ロット・到着済みのみ）

仕入れごとに「ロット」として単価・残個数を保持します。出荷時は古いロットから順に消費されます（先入先出法）。未到着ぶんは灰色で参考表示し、在庫数には含めません。

セット/販売単位の在庫可能数

構成品の在庫から、あと何セット作れるかを表示します。

在庫調整（手動）

在庫を手動で増減します。種類ごとに扱いが変わります：
・私用：在庫を先入先出で引き、原価も消費（売上には計上しない）
・廃棄：在庫を引く（廃棄ロスとして記録／売上・原価消費には含めない）
・棚卸で減らす：在庫を引き「販売」として計上（販路・販売単価を指定可）
・棚卸で増やす：新規ロットで在庫を追加（原価は平均在庫評価額）

👥 チーム共有（クラウド保存）

Google Apps Script＋Googleドライブを使い、チーム全員で同じ在庫データを共有します。接続すると、変更が自動でクラウドに保存され、他メンバーの画面にも反映されます（約15秒ごとに自動取得）。未接続ならこの端末内だけの保存です。
注意：ほぼ同時刻に別々の人が編集すると、後から保存した方で上書きされます。同じ商品を同時に触らない・こまめに「最新取得」を心がけてください。

▶ セットアップ手順（管理者が1回だけ・コード付き）
  1. 管理者が script.google.com で新規プロジェクトを作成し、下の「コードをコピー」で全文を貼り付け。
  2. 「デプロイ → 新しいデプロイ → 種類:ウェブアプリ」。実行するユーザー=自分、アクセスできるユーザー=全員 でデプロイ。初回はGoogleドライブへの権限承認が出るので「許可」。
  3. 表示された /exec のURLを上の欄に貼り、「保存して接続」。管理者の現在のデータがクラウドに保存されます。
  4. 各メンバーは同じアプリ(index.html)を開き、同じURLを入力して「保存して接続」。以後は全員が同じデータを共有します。
  5. （便利）配布リンクの末尾に ?cloud=（execのURL） を付けておくと、開くだけで自動接続されます。例：https://xxx.netlify.app/?cloud=https://script.google.com/macros/s/AAA/exec
var FILE_NAME = 'inventory-shared-db.json';

function _getFile(){
  var it = DriveApp.getFilesByName(FILE_NAME);
  if (it.hasNext()) return it.next();
  return DriveApp.createFile(FILE_NAME, JSON.stringify({ rev: 0, data: '' }), MimeType.PLAIN_TEXT);
}

function doGet(e){
  var f = _getFile();
  var obj;
  try { obj = JSON.parse(f.getBlob().getDataAsString()); } catch (err) { obj = { rev: 0, data: '' }; }
  var out = JSON.stringify(obj);
  var cb = e && e.parameter && e.parameter.callback;
  if (cb) return ContentService.createTextOutput(cb + '(' + out + ')').setMimeType(ContentService.MimeType.JAVASCRIPT);
  return ContentService.createTextOutput(out).setMimeType(ContentService.MimeType.JSON);
}

function doPost(e){
  var data = (e && e.parameter && e.parameter.data) || '';
  var rev = (e && e.parameter && e.parameter.rev) || Date.now();
  var f = _getFile();
  f.setContent(JSON.stringify({ rev: Number(rev), data: data }));
  return ContentService.createTextOutput('OK');
}

LINE送信の設定

発注の「出荷を登録して送信」で使うLINE送信方法を選べます。通常のLINEはLINEアプリが開いて送信相手を選ぶ方式（設定不要）。公式LINEはGoogle Apps Script中継で自動送信する方式です（下の設定が必要）。

↓ 公式LINEを選んだ場合のみ、以下のApps Script連携が必要です。channel ID / secret は Apps Script 内にだけ保管され、このアプリには出ません。

▶ セットアップ手順（Apps Scriptコード付き）
  1. LINE DevelopersでMessaging APIチャネルを作成/選択し、channel ID と channel secret を控える。
  2. script.google.com で新規プロジェクトを作成し、下のコードを貼り付け、CHANNEL_ID・CHANNEL_SECRET を入力。全友だちに送るなら TO_USER_ID は空のまま。特定の相手だけに送るなら、その人のuserIdを入れる。
  3. 「デプロイ → 新しいデプロイ → 種類:ウェブアプリ」。実行するユーザー=自分、アクセスできるユーザー=全員 にしてデプロイ。
  4. 表示された /exec のURLを上の欄に貼って「保存」→「テスト送信」。LINEに届けば完了。
const CHANNEL_ID = 'ここにchannel ID';
const CHANNEL_SECRET = 'ここにchannel secret';
const TO_USER_ID = ''; // 空=全友だちにbroadcast / 指定=そのuserIdにpush

function doPost(e){ return handle_(e); }
function doGet(e){ return handle_(e); }

function handle_(e){
  var text = (e && e.parameter && e.parameter.text) || 'テスト送信';
  var token = getToken_();
  var url, payload;
  if (TO_USER_ID) {
    url = 'https://api.line.me/v2/bot/message/push';
    payload = { to: TO_USER_ID, messages: [{ type:'text', text: text }] };
  } else {
    url = 'https://api.line.me/v2/bot/message/broadcast';
    payload = { messages: [{ type:'text', text: text }] };
  }
  var res = UrlFetchApp.fetch(url, {
    method: 'post', contentType: 'application/json',
    headers: { Authorization: 'Bearer ' + token },
    payload: JSON.stringify(payload), muteHttpExceptions: true });
  return ContentService.createTextOutput('LINE ' + res.getResponseCode());
}

function getToken_(){
  var res = UrlFetchApp.fetch('https://api.line.me/v2/oauth/accessToken', {
    method: 'post',
    payload: { grant_type: 'client_credentials', client_id: CHANNEL_ID, client_secret: CHANNEL_SECRET },
    muteHttpExceptions: true });
  return JSON.parse(res.getContentText()).access_token;
}
